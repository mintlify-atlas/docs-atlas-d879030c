The `kamal secrets` command provides utilities for working with secrets stored in external vaults like 1Password, AWS Secrets Manager, or other secret management systems.

## Overview

Kamal's secret helpers allow you to:
- Fetch secrets from vault systems
- Extract individual secret values
- Debug secret loading

## Commands

### kamal secrets fetch

Fetch secrets from your configured vault:

```bash
kamal secrets fetch [SECRETS...]
```

**Examples:**

```bash
# Fetch all secrets
kamal secrets fetch

# Fetch specific secrets
kamal secrets fetch DB_PASSWORD API_KEY
```

### kamal secrets extract

Extract a single secret from the results of a fetch call:

```bash
kamal secrets extract SECRET_NAME
```

This is typically used in scripts or as part of command substitution.

**Example:**

```bash
# Use in command substitution
DB_PASS=$(kamal secrets fetch DB_PASSWORD | kamal secrets extract DB_PASSWORD)
echo "Database password: $DB_PASS"
```

### kamal secrets print

Print the secrets for debugging (useful for verifying secret loading):

```bash
kamal secrets print
```

<Warning>
  This command outputs secrets in plain text. Use only in secure environments for debugging.
</Warning>

## Vault Adapters

Kamal supports multiple vault systems through adapters. Configure your vault in `.kamal/secrets`:

### 1Password

```bash .kamal/secrets
# Load from 1Password CLI
KAMAL_REGISTRY_PASSWORD=$(op read "op://vault/item/password")
RAILS_MASTER_KEY=$(op read "op://vault/rails/master_key")
```

### AWS Secrets Manager

```bash .kamal/secrets
# Load from AWS Secrets Manager
KAMAL_REGISTRY_PASSWORD=$(aws secretsmanager get-secret-value --secret-id prod/registry/password --query SecretString --output text)
DB_PASSWORD=$(aws secretsmanager get-secret-value --secret-id prod/db/password --query SecretString --output text)
```

### HashiCorp Vault

```bash .kamal/secrets
# Load from Vault
KAMAL_REGISTRY_PASSWORD=$(vault kv get -field=password secret/registry)
DB_PASSWORD=$(vault kv get -field=password secret/database)
```

### Environment Variables

For simpler setups, use environment variables directly:

```bash .kamal/secrets
KAMAL_REGISTRY_PASSWORD=$REGISTRY_TOKEN
RAILS_MASTER_KEY=$RAILS_KEY
```

## Configuration

Secrets are loaded from `.kamal/secrets` file during Kamal commands. The file is executed as a shell script, allowing command substitution.

<Info>
  The `.kamal/secrets` file should never be committed to version control. Add it to `.gitignore`.
</Info>

## Use Cases

<Accordion title="Verify secret loading">
  Check that secrets are being loaded correctly:
  
  ```bash
  kamal secrets print
  ```
  
  This shows all secrets that will be available to your deployment.
</Accordion>

<Accordion title="Fetch specific secrets">
  Retrieve individual secrets for use in scripts:
  
  ```bash
  # Fetch registry password
  REGISTRY_PASS=$(kamal secrets fetch KAMAL_REGISTRY_PASSWORD | kamal secrets extract KAMAL_REGISTRY_PASSWORD)
  
  # Use in docker login
  echo $REGISTRY_PASS | docker login registry.example.com -u myuser --password-stdin
  ```
</Accordion>

<Accordion title="CI/CD integration">
  Use secrets commands in CI pipelines:
  
  ```yaml .github/workflows/deploy.yml
  - name: Deploy with Kamal
    env:
      KAMAL_REGISTRY_PASSWORD: ${{ secrets.REGISTRY_PASSWORD }}
    run: |
      kamal deploy
  ```
</Accordion>

<Accordion title="Debug secret issues">
  When secrets aren't loading correctly:
  
  ```bash
  # Print all secrets (redacted by default)
  kamal secrets print
  
  # Check if specific secret exists
  kamal secrets fetch MY_SECRET
  
  # Verify vault CLI is working
  op whoami  # for 1Password
  aws sts get-caller-identity  # for AWS
  vault status  # for HashiCorp Vault
  ```
</Accordion>

## Security Best Practices

<Warning>
  **Never commit secrets to version control**
  
  - Add `.kamal/secrets` to `.gitignore`
  - Use vault systems for production secrets
  - Rotate secrets regularly
  - Use CI/CD secret management for automated deployments
</Warning>

<Tip>
  **Create a secrets example file**
  
  ```bash
  # Copy your secrets file to create an example
  cp .kamal/secrets .kamal/secrets.example
  
  # Replace actual values with placeholders
  sed -i 's/=.*/=your-secret-here/' .kamal/secrets.example
  
  # Commit the example (not the real secrets)
  git add .kamal/secrets.example
  ```
</Tip>

## Troubleshooting

<Accordion title="Secrets not loading">
  **Issue**: Secrets aren't being loaded during deployment
  
  **Solution**:
  - Verify `.kamal/secrets` file exists and is executable
  - Check vault CLI is installed and authenticated
  - Test command substitution manually:
    ```bash
    source .kamal/secrets
    echo $KAMAL_REGISTRY_PASSWORD
    ```
</Accordion>

<Accordion title="Command substitution failing">
  **Issue**: Vault commands in `.kamal/secrets` return errors
  
  **Solution**:
  - Test vault command directly:
    ```bash
    op read "op://vault/item/field"
    ```
  - Check authentication: `op signin`
  - Verify secret path is correct
  - Ensure vault CLI is in PATH
</Accordion>

<Accordion title="Secrets showing in logs">
  **Issue**: Secret values appear in deployment logs
  
  **Solution**: Kamal automatically redacts secret values. If you see secrets:
  - They may be printed by your application
  - Check custom hooks aren't echoing secrets
  - Avoid using `set -x` in scripts that handle secrets
</Accordion>
