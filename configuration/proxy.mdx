---
title: Proxy Configuration
description: Configure kamal-proxy for zero-downtime deployments and SSL
---

Kamal uses [kamal-proxy](https://github.com/basecamp/kamal-proxy) to provide zero-downtime deployments. It runs on ports 80 and 443 and forwards requests to your application containers.

## Basic Configuration

```yaml
proxy:
  ssl: true
  host: app.example.com
```

Proxy options are application-specific and are set when deploying the app, not when booting the proxy container.

## Hosts

<ParamField path="host" type="string">
  Single hostname for the application.
</ParamField>

<ParamField path="hosts" type="array">
  Multiple hostnames for the application.
</ParamField>

```yaml
proxy:
  # Single host
  host: app.example.com
  
  # Or multiple hosts
  hosts:
    - app.example.com
    - www.app.example.com
```

<Note>
If no hosts are set, all requests are forwarded to your app (except requests matching other apps with hosts configured).
</Note>

## App Port

<ParamField path="app_port" type="integer" default="80">
  The port your application container listens on.
</ParamField>

```yaml
proxy:
  host: app.example.com
  app_port: 3000
```

## SSL Configuration

### Automatic SSL (Let's Encrypt)

<ParamField path="ssl" type="boolean" default="false">
  Enable automatic HTTPS via Let's Encrypt.
</ParamField>

```yaml
proxy:
  ssl: true
  host: app.example.com
```

**Requirements:**
- Single server deployment
- `host` option must be set
- Host DNS must point to the server
- Port 443 must be open

<Warning>
With `ssl: true`, kamal-proxy will not forward `X-Forwarded-For` and `X-Forwarded-Proto` headers by default unless you set `forward_headers: true`.
</Warning>

### Custom SSL Certificates

For multi-server deployments or existing certificates:

```yaml
proxy:
  ssl:
    certificate_pem: CERTIFICATE_PEM
    private_key_pem: PRIVATE_KEY_PEM
  hosts:
    - app.example.com
    - www.app.example.com
```

Certificates are read from secrets in `.kamal/secrets`:

```shell
# .kamal/secrets
CERTIFICATE_PEM=$(cat /path/to/certificate.pem)
PRIVATE_KEY_PEM=$(cat /path/to/private-key.pem)
```

<Note>
Custom certificates allow SSL on multiple servers, which is not possible with Let's Encrypt.
</Note>

### SSL Redirect

<ParamField path="ssl_redirect" type="boolean" default="true">
  Redirect HTTP requests to HTTPS when SSL is enabled.
</ParamField>

```yaml
proxy:
  ssl: true
  host: app.example.com
  ssl_redirect: false  # Allow HTTP traffic through
```

### Forward Headers

<ParamField path="forward_headers" type="boolean">
  Forward `X-Forwarded-For` and `X-Forwarded-Proto` headers.
</ParamField>

```yaml
proxy:
  ssl: true
  host: app.example.com
  forward_headers: true
```

**Default behavior:**
- With `ssl: true`: headers are NOT forwarded
- With `ssl: false`: headers ARE forwarded

<Note>
Only set `forward_headers: true` if you trust the proxy in front of kamal-proxy.
</Note>

## Timeouts

<ParamField path="response_timeout" type="integer" default="30">
  Seconds to wait for requests to complete before timing out.
</ParamField>

```yaml
proxy:
  host: app.example.com
  response_timeout: 60  # 60 seconds for slow requests
```

<ParamField path="deploy_timeout" type="integer" default="30">
  Seconds to wait for new containers to become healthy during deployment.
</ParamField>

<ParamField path="drain_timeout" type="integer" default="30">
  Seconds to wait for connections to drain when stopping containers.
</ParamField>

```yaml
# Root-level timeouts affect proxy behavior
deploy_timeout: 60
drain_timeout: 45
```

## Health Checks

<ParamField path="healthcheck" type="object">
  Configure health check behavior during deployments.
</ParamField>

```yaml
proxy:
  healthcheck:
    path: /health
    interval: 3
    timeout: 5
```

<ParamField path="healthcheck.path" type="string" default="/up">
  Health check endpoint path.
</ParamField>

<ParamField path="healthcheck.interval" type="integer" default="1">
  Seconds between health check requests.
</ParamField>

<ParamField path="healthcheck.timeout" type="integer" default="5">
  Seconds to wait for each health check response.
</ParamField>

<Note>
Health checks only run during deployment. Once the app is up, the proxy stops hitting the endpoint.
</Note>

## Path-Based Routing

Route requests based on URL path prefixes:

<ParamField path="path_prefix" type="string">
  Comma-separated list of path prefixes.
</ParamField>

<ParamField path="path_prefixes" type="array">
  Array of path prefixes.
</ParamField>

<Tabs>
  <Tab title="String Format">
    ```yaml
    proxy:
      host: app.example.com
      path_prefix: /api,/oauth_callback
    ```
  </Tab>
  
  <Tab title="Array Format">
    ```yaml
    proxy:
      host: app.example.com
      path_prefixes:
        - /api
        - /oauth_callback
        - /webhooks
    ```
  </Tab>
</Tabs>

### Strip Path Prefix

<ParamField path="strip_path_prefix" type="boolean" default="true">
  Remove the path prefix before forwarding to the app.
</ParamField>

```yaml
proxy:
  path_prefix: /api
  strip_path_prefix: false
```

**With `strip_path_prefix: true` (default):**
- Request: `https://app.example.com/api/users/123`
- Forwarded: `/users/123`

**With `strip_path_prefix: false`:**
- Request: `https://app.example.com/api/users/123`
- Forwarded: `/api/users/123`

## Buffering

<ParamField path="buffering" type="object">
  Configure request and response buffering.
</ParamField>

```yaml
proxy:
  buffering:
    requests: true
    responses: true
    max_request_body: 40_000_000   # 40 MB
    max_response_body: 0           # No limit
    memory: 2_000_000              # 2 MB
```

<ParamField path="buffering.requests" type="boolean" default="true">
  Buffer request bodies.
</ParamField>

<ParamField path="buffering.responses" type="boolean" default="true">
  Buffer response bodies.
</ParamField>

<ParamField path="buffering.max_request_body" type="integer" default="1_000_000_000">
  Maximum request body size in bytes (1 GB default).
</ParamField>

<ParamField path="buffering.max_response_body" type="integer" default="0">
  Maximum response body size in bytes (0 = unlimited).
</ParamField>

<ParamField path="buffering.memory" type="integer" default="1_000_000">
  Memory limit for buffering in bytes. Larger buffers use disk (1 MB default).
</ParamField>

## Logging

<ParamField path="logging" type="object">
  Configure proxy request/response logging.
</ParamField>

```yaml
proxy:
  logging:
    request_headers:
      - Cache-Control
      - User-Agent
      - X-Request-ID
    response_headers:
      - X-Request-ID
      - X-Runtime
```

<ParamField path="logging.request_headers" type="array">
  Request headers to log. Default: `[Cache-Control, Last-Modified, User-Agent]`
</ParamField>

<ParamField path="logging.response_headers" type="array">
  Response headers to log.
</ParamField>

## Proxy Container Configuration

Configure the kamal-proxy container itself:

```yaml
proxy:
  run:
    http_port: 8080
    https_port: 8443
    metrics_port: 9090
    debug: true
    log_max_size: "30m"
    publish: false
    bind_ips:
      - 0.0.0.0
    registry: registry.example.com
    repository: custom/kamal-proxy
    version: v0.8.0
    options:
      memory: 512m
      cpus: 0.5
      label:
        - custom.label=kamal-proxy
```

<ParamField path="run.http_port" type="integer" default="80">
  HTTP port for the proxy.
</ParamField>

<ParamField path="run.https_port" type="integer" default="443">
  HTTPS port for the proxy.
</ParamField>

<ParamField path="run.metrics_port" type="integer">
  Port for Prometheus metrics.
</ParamField>

<ParamField path="run.debug" type="boolean" default="false">
  Enable debug logging.
</ParamField>

<ParamField path="run.log_max_size" type="string" default="10m">
  Maximum log file size.
</ParamField>

<ParamField path="run.publish" type="boolean" default="true">
  Publish ports to the host.
</ParamField>

<ParamField path="run.bind_ips" type="array">
  IP addresses to bind when publishing ports.
</ParamField>

<ParamField path="run.registry" type="string">
  Registry for the kamal-proxy image (defaults to Docker Hub).
</ParamField>

<ParamField path="run.repository" type="string" default="basecamp/kamal-proxy">
  Repository for the kamal-proxy image.
</ParamField>

<ParamField path="run.version" type="string">
  Version tag of the kamal-proxy image.
</ParamField>

<ParamField path="run.options" type="object">
  Additional Docker run options.
</ParamField>

## Enabling/Disabling Proxy on Roles

The proxy is enabled by default on the primary role and disabled on others.

### Disable on Primary Role

```yaml
servers:
  web:
    hosts:
      - 192.168.0.1
    proxy: false
```

### Enable on Non-Primary Role

<Tabs>
  <Tab title="Inherit Config">
    ```yaml
    servers:
      api:
        hosts:
          - 192.168.0.2
        proxy: true  # Uses root proxy config
    ```
  </Tab>
  
  <Tab title="Custom Config">
    ```yaml
    servers:
      api:
        hosts:
          - 192.168.0.2
        proxy:
          host: api.example.com
          ssl: true
          app_port: 3000
    ```
  </Tab>
</Tabs>

## Complete Examples

<Accordion title="Production with SSL">
```yaml
service: my-app
image: my-user/my-app

servers:
  web:
    - 192.168.0.1

proxy:
  ssl: true
  host: app.example.com
  app_port: 3000
  response_timeout: 60
  
  healthcheck:
    path: /health
    interval: 2
    timeout: 5
  
  logging:
    request_headers:
      - User-Agent
      - X-Request-ID
    response_headers:
      - X-Request-ID
      - X-Runtime
```
</Accordion>

<Accordion title="Multi-Server with Custom Certificates">
```yaml
servers:
  web:
    hosts:
      - web-1.example.com
      - web-2.example.com

proxy:
  ssl:
    certificate_pem: SSL_CERTIFICATE
    private_key_pem: SSL_PRIVATE_KEY
  hosts:
    - app.example.com
    - www.app.example.com
  app_port: 3000
```
</Accordion>

<Accordion title="API with Path Routing">
```yaml
servers:
  api:
    hosts:
      - api.example.com
    proxy:
      host: api.example.com
      ssl: true
      path_prefixes:
        - /v1
        - /v2
        - /oauth
      strip_path_prefix: false
      app_port: 4000
```
</Accordion>

<Accordion title="Custom Proxy Container">
```yaml
proxy:
  ssl: true
  host: app.example.com
  
  run:
    http_port: 8080
    https_port: 8443
    metrics_port: 9090
    debug: true
    version: v0.8.0
    options:
      memory: 1g
      cpus: 1
```
</Accordion>

## Managing the Proxy

```bash
# Boot the proxy
kamal proxy boot

# Reboot the proxy
kamal proxy reboot

# Stop the proxy
kamal proxy stop

# View proxy logs
kamal proxy logs
```
