title: Setting Up Accessories

Accessories are additional services like databases, Redis, or message queues that run alongside your application. Kamal makes it easy to manage these services as Docker containers.

## What Are Accessories?

Accessories are separate Docker containers that provide supporting services for your application. Unlike your main application, accessories:

- Run from pre-built images (like `mysql:8.0` or `redis:7`)
- Persist data between deployments
- Can be managed independently
- Run on dedicated servers

## Database Setup

### MySQL Configuration

<Steps>

<Step title="Configure in deploy.yml">
```yaml
accessories:
  db:
    image: mysql:8.0
    host: 192.168.0.2
    port: 3306
    env:
      clear:
        MYSQL_ROOT_HOST: '%'
      secret:
        - MYSQL_ROOT_PASSWORD
    directories:
      - data:/var/lib/mysql
```
</Step>

<Step title="Set secrets">
Add the password to `.kamal/secrets`:
```bash
MYSQL_ROOT_PASSWORD=your-secure-password-here
```
</Step>

<Step title="Boot the accessory">
```bash
kamal accessory boot db
```
</Step>

</Steps>

## Redis Setup

### Basic Redis Configuration

```yaml
accessories:
  redis:
    image: redis:7
    host: 192.168.0.2
    port: 6379
    directories:
      - data:/data
```

### Redis with Password

```yaml
accessories:
  redis:
    image: redis:7
    host: 192.168.0.2
    port: 6379
    cmd: redis-server --requirepass "${REDIS_PASSWORD}"
    env:
      secret:
        - REDIS_PASSWORD
    directories:
      - data:/data
```

### Valkey (Redis Alternative)

Valkey is a Redis-compatible fork:

```yaml
accessories:
  redis:
    image: valkey/valkey:8
    host: 192.168.0.2
    port: 6379
    directories:
      - data:/data
```

## Configuration Options

### Required Fields

Every accessory needs:

```yaml
accessories:
  service_name:
    image: docker/image:tag  # Required
    host: 192.168.0.2        # Required
```

### Port Mapping

Expose ports to the host:

```yaml
accessories:
  db:
    image: postgres:16
    host: 192.168.0.2
    port: 5432  # Maps container port 5432 to host port 5432
```

Custom port mapping:

```yaml
accessories:
  db:
    image: postgres:16
    host: 192.168.0.2
    port: "5433:5432"  # Host port 5433 -> container port 5432
```

### Environment Variables

Set environment variables for configuration:

```yaml
accessories:
  db:
    image: mysql:8.0
    host: 192.168.0.2
    env:
      clear:  # Visible in container inspect
        MYSQL_DATABASE: myapp_production
        MYSQL_ROOT_HOST: '%'
      secret:  # Loaded from .kamal/secrets
        - MYSQL_ROOT_PASSWORD
        - MYSQL_USER
        - MYSQL_PASSWORD
```

### Data Persistence

Use directories for persistent data:

```yaml
accessories:
  db:
    image: postgres:16
    host: 192.168.0.2
    directories:
      - data:/var/lib/postgresql/data  # Named volume
      - backups:/backups                # Separate volume for backups
```

From `lib/kamal/cli/accessory.rb:67-71`, Kamal creates directories with proper permissions:

```ruby
accessory.directories.each do |(local, config)|
  execute *accessory.make_directory(local)
  execute :chmod, config[:mode], local if config[:mode]
  execute :chown, config[:owner], local if config[:owner]
end
```

### File Uploads

Upload configuration files to the container:

```yaml
accessories:
  db:
    image: mysql:8.0
    host: 192.168.0.2
    files:
      - config/mysql/production.cnf:/etc/mysql/my.cnf
      - db/init.sql:/docker-entrypoint-initdb.d/setup.sql
    directories:
      - data:/var/lib/mysql
```

File options with permissions:

```yaml
accessories:
  app:
    image: custom/app:latest
    host: 192.168.0.2
    files:
      - path: config/app.conf
        host_path: /etc/app/app.conf
        mode: "0644"
        owner: "1000:1000"
```

### Custom Commands

Override the default container command:

```yaml
accessories:
  redis:
    image: redis:7
    host: 192.168.0.2
    cmd: redis-server --appendonly yes --requirepass "${REDIS_PASSWORD}"
```

## Managing Accessories

### Boot Accessories

```bash
# Boot a specific accessory
kamal accessory boot db

# Boot all accessories
kamal accessory boot all
```

From `lib/kamal/cli/accessory.rb:6-40`, booting:
- Creates necessary directories
- Uploads configuration files
- Uploads secrets
- Starts the container

### Reboot Accessories

```bash
# Reboot a specific accessory (pull image, stop, remove, boot)
kamal accessory reboot db

# Reboot all accessories
kamal accessory reboot all
```

### Start/Stop Accessories

```bash
# Stop accessory
kamal accessory stop redis

# Start accessory
kamal accessory start redis

# Restart accessory (stop then start)
kamal accessory restart redis
```

### View Accessory Details

```bash
# Show details for specific accessory
kamal accessory details db

# Show all accessories
kamal accessory details all
```

### Accessory Logs

```bash
# View recent logs
kamal accessory logs db

# Follow logs in real-time
kamal accessory logs db --follow

# Show last 100 lines
kamal accessory logs db --lines 100

# Search logs
kamal accessory logs db --grep "ERROR"
```

### Execute Commands

```bash
# Run a command in the accessory container
kamal accessory exec db "mysql -u root -p\${MYSQL_ROOT_PASSWORD}"

# Interactive shell
kamal accessory exec db --interactive --reuse "bash"
```

### Remove Accessories

```bash
# Remove accessory (stops, removes container, image, and data)
kamal accessory remove db --confirmed

# Remove all accessories
kamal accessory remove all --confirmed
```

<Warning>
Removing an accessory deletes all its data. Backup first!
</Warning>

## Multiple Accessory Hosts

### Same Accessory on Multiple Servers

```yaml
accessories:
  redis:
    image: redis:7
    hosts:
      - 192.168.0.2
      - 192.168.0.3
    port: 6379
    directories:
      - data:/data
```

Kamal will create a Redis instance on both hosts.

### Different Accessories on Different Hosts

```yaml
accessories:
  db:
    image: postgres:16
    host: 192.168.0.2
    directories:
      - data:/var/lib/postgresql/data
  
  redis:
    image: redis:7
    host: 192.168.0.3
    directories:
      - data:/data
```

## Connecting Your App

### Environment Variables for Connection

Set connection details in your app configuration:

```yaml
servers:
  web:
    hosts:
      - 192.168.0.1
    env:
      clear:
        DATABASE_HOST: 192.168.0.2
        DATABASE_PORT: 3306
        REDIS_HOST: 192.168.0.2
        REDIS_PORT: 6379
      secret:
        - DATABASE_PASSWORD
        - REDIS_PASSWORD

accessories:
  db:
    image: mysql:8.0
    host: 192.168.0.2
    port: 3306
    env:
      secret:
        - MYSQL_ROOT_PASSWORD
    directories:
      - data:/var/lib/mysql
  
  redis:
    image: redis:7
    host: 192.168.0.2
    port: 6379
    directories:
      - data:/data
```

### Database URL Format

For Rails and other frameworks:

```bash
# .kamal/secrets
DATABASE_URL=mysql2://root:${MYSQL_ROOT_PASSWORD}@192.168.0.2:3306/myapp_production
REDIS_URL=redis://192.168.0.2:6379/0
```

## Backup Strategies

### Database Backups

Create a backup script and run it via cron:

```bash
# On the database host
kamal accessory exec db "mysqldump -u root -p\${MYSQL_ROOT_PASSWORD} myapp_production" > backup.sql
```

Or create a backup accessory:

```yaml
accessories:
  db-backup:
    image: mysql:8.0
    host: 192.168.0.2
    cmd: |
      sh -c 'while true; do
        mysqldump -h db -u root -p$${MYSQL_ROOT_PASSWORD} myapp_production > /backups/backup-$$(date +%Y%m%d-%H%M%S).sql
        sleep 86400
      done'
    env:
      secret:
        - MYSQL_ROOT_PASSWORD
    directories:
      - backups:/backups
```

### Redis Persistence

Enable RDB snapshots:

```yaml
accessories:
  redis:
    image: redis:7
    host: 192.168.0.2
    cmd: redis-server --save 60 1 --appendonly yes
    directories:
      - data:/data
```

## Troubleshooting

<Accordion title="Accessory fails to start">
Check logs:
```bash
kamal accessory logs db --lines 100
```

Common issues:
- Port already in use
- Insufficient memory
- Missing secrets
- Data directory permissions
</Accordion>

<Accordion title="Cannot connect to accessory from app">
Verify:
1. Network connectivity:
   ```bash
   kamal app exec --reuse "ping 192.168.0.2"
   ```
2. Port is exposed:
   ```bash
   kamal accessory details db
   ```
3. Firewall rules allow traffic
4. Environment variables are correct
</Accordion>

<Accordion title="Data lost after reboot">
Ensure directories are configured:
```yaml
directories:
  - data:/var/lib/mysql  # Must mount data directory
```

Check directory exists on host:
```bash
kamal accessory exec db "ls -la /var/lib/mysql"
```
</Accordion>

<Accordion title="Slow database performance">
Tune configuration:
```yaml
accessories:
  db:
    image: mysql:8.0
    host: 192.168.0.2
    options:
      memory: 2g
      cpus: 2
    files:
      - config/mysql/my.cnf:/etc/mysql/my.cnf
```
</Accordion>

## Best Practices

1. **Separate servers** - Run accessories on dedicated servers, not with your app
2. **Persist data** - Always configure directories for data persistence
3. **Use secrets** - Store passwords in `.kamal/secrets`, never in deploy.yml
4. **Regular backups** - Implement automated backup strategies
5. **Monitor resources** - Set memory and CPU limits
6. **Update regularly** - Keep accessory images updated with security patches
7. **Test restores** - Regularly test your backup restoration process
8. **Document versions** - Pin specific image versions for stability
