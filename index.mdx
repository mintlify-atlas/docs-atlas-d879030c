# Deploy web apps anywhere

Kamal lets you deploy web applications to any server running Docker with zero downtime. Built by Basecamp, Kamal uses [kamal-proxy](https://github.com/basecamp/kamal-proxy) to seamlessly switch requests between containers during deployments.

## How It Works

Kamal orchestrates Docker-based deployments through a simple workflow:

<Steps>
  <Step title="Configure your deployment">
    Create a `config/deploy.yml` file defining your application, servers, and deployment settings
  </Step>
  <Step title="Build and push">
    Kamal builds your Docker image and pushes it to your container registry
  </Step>
  <Step title="Deploy with zero downtime">
    Kamal deploys containers to your servers and uses kamal-proxy to route traffic seamlessly
  </Step>
</Steps>

## Quick Example

Deploy a containerized web application in just a few steps:

```bash Terminal
# Install Kamal
gem install kamal

# Initialize configuration
kamal init

# Set up servers and deploy
kamal setup
```

Your application is now running with zero-downtime deployment capabilities!

## What You Can Deploy

Kamal works with any application that can be containerized with Docker:

- Ruby on Rails applications
- Node.js and Express servers
- Python Django or Flask apps
- Go web services
- Static sites with Nginx
- Any other Docker-compatible application
